## 1.2 Arm manipulation agent: IK-based controller

Arm agent 不需要先做神经网络训练，它是一个 IK-based controller。

输入：

```python
action_chunk
h
```

其中：

```python
action_chunk = Δx_{t:t+H}^{ee,b}
             = { (Δp_{t,k}^{ee,b}, Δr_{t,k}^{ee,b}) }_{k=1}^{H}
```

也就是说，action chunk 是一段未来末端位姿增量序列。

每个 step 的 arm action 维度是 6：

```python
Δx_ee = [
    Δp_x,
    Δp_y,
    Δp_z,
    Δr_roll,
    Δr_pitch,
    Δr_yaw
]
```

`h ∈ {1, 2, 3}` 表示每次高层输出 action chunk 之后，底层实际执行多少个 chunk step，再重新请求高层输出。

请实现：

```python
ArmIKController
```

其功能：

1. 接收当前 ee pose。
2. 接收 action_chunk。
3. 根据 h 依次执行前 h 个 chunk step。
4. 对每个目标 ee pose 使用 IK 求解 arm joint target。
5. 加入 joint limit clipping / safety clipping。
6. 返回 arm joint target。
7. 如果仓库已有 IK，请复用已有 IK；如果没有，请实现一个接口类，并明确 TODO，不要写死某个机器人模型。

建议 IK 优化目标形式：

```python
min_q || FK(q) - x_target ||^2
    + λ_smooth || q - q_prev ||^2
    + λ_limit joint_limit_penalty(q)
    - λ_manip * manipulability(q)
```

如果当前工程短期只支持已有 IK，那么先实现可运行版本，不要强行完整实现复杂 IK。

## 2.1 arm -> base message: m_a2b

定义：

```python
m_a2b = [
    Δp_bar_ee,
    A_ee,
    eta_manip
]
```

其中：

### 机械臂未来短时间平均运动方向

```python
Δp_bar_ee = (1 / H) * sum_{k=1}^{H} Δp_{t,k}^{ee}
```

维度是 3。

### 机械臂未来短时间动作幅度

```python
A_ee = sum_{k=1}^{H} || Δp_{t,k}^{ee} ||_2
```

维度是 1。

### 当前机械臂可操作性指标

```python
eta_manip = sqrt(det(J_a @ J_a.T))
```

维度是 1。

为了数值稳定，请实现成：

```python
JJT = J_a @ J_a.transpose(-1, -2)
eta_manip = sqrt(clamp(det(JJT + eps * I), min=0.0))
```

如果 J 不是满秩，或者 det 数值不稳定，可以改用：

```python
eta_manip = sqrt(prod(clamp(singular_values(J_a), min=eps)))
```

请把这一点写清楚，并保证 batch 版本可运行。

最终：

```python
m_a2b.shape == [B, 5]
```

其中 5 = 3 + 1 + 1。

---

## 2.2 arm -> gripper message: m_a2g

定义：

```python
m_a2g = [
    rho_align,
    terminal_ee_motion
]
```

其中：

### arm-gripper 对齐评分

```python
rho_align = exp(- ||p_obj_w||_2^2 / sigma_p^2)
```

`p_obj_w` 表示物体相对于 wrist camera 的位置。

### 末端当前动作幅度

```python
terminal_ee_motion = || Δp_{t,H}^{ee} ||_2
```

最终：

```python
m_a2g.shape == [B, 2]
```

# 4. 模型结构建议

请实现或修改成如下结构：

```python
class MultiAgentHighLevelPolicy(nn.Module):
    def __init__(...):
        self.encoder = ...
        self.shared_backbone = ...
        self.base_head = ...
        self.arm_chunk_head = ...
        self.grasp_intent_head = ...
        self.h_head = ...

    def forward(obs):
        feat = self.encoder(obs)
        z = self.shared_backbone(feat)

        base_command = self.base_head(z)
        arm_action_chunk = self.arm_chunk_head(z).view(B, H, 6)
        grasp_intent_logit = self.grasp_intent_head(z)
        h_logits = self.h_head(z)

        return {
            "base_command": base_command,
            "arm_action_chunk": arm_action_chunk,
            "grasp_intent_logit": grasp_intent_logit,
            "h_logits": h_logits
        }
```

如果当前代码已经有 actor-critic，请不要完全推翻它。请把 actor head 改造成多头输出，critic 继续复用原有 value head。

如果当前策略是 flat actor 输出，请至少做到：

1. 内部多头输出；
2. 最后 flatten 成原框架需要的 action；
3. 在 env step 前 unflatten 回结构化 action。

## Stage 1: Base locomotion agent 训练

目标：训练 base policy 跟踪 command，并学会利用 `m_a2b` 处理机械臂运动扰动。

输入：

```python
robot_state + base_command + m_a2b
```

输出：

```python
leg_joint_target
```

训练方法：

```python
PPO
```

奖励建议：

```python
r_base =
    w_v * velocity_tracking
  + w_yaw * yaw_tracking
  + w_height * height_tracking
  + w_pitch * pitch_tracking
  - w_energy * torque_penalty
  - w_smooth * action_smoothness_penalty
  - w_slip * foot_slip_penalty
  - w_fall * fall_penalty
```

额外加入 arm-base coordination reward：

```python
r_assist =
    I(eta_manip < eta_threshold)
    * I(A_ee > A_threshold)
    * max(0, dot(v_base_xy, normalize(Δp_bar_ee_xy)))
```

含义：当机械臂接近奇异位形，并且未来动作幅度较大时，base 应该往有利于机械臂运动的方向移动。

请添加 config 开关：
