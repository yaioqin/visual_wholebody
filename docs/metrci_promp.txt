============================================================
一、指标来源和定义说明
============================================================

我希望你参考两类 whole-body loco-manipulation 论文中的评估方式。

------------------------------------------------------------
1. RoboDuet 类指标
------------------------------------------------------------

RoboDuet 评估 whole-body legged loco-manipulation 时，主要使用以下指标：

A. Velocity Tracking Error
定义：
机器人实际 base velocity 和 command velocity 之间的误差。

需要评估：
- vx tracking error
- vy tracking error
- yaw angular velocity tracking error

公式：
vx_err = abs(actual_vx - command_vx)
vy_err = abs(actual_vy - command_vy)
yaw_err = abs(actual_yaw_rate - command_yaw_rate)

其中：
- actual_vx, actual_vy 来自 base frame 下的 base linear velocity
- actual_yaw_rate 来自 base frame 下的 base angular velocity z 分量
- command_vx, command_vy, command_yaw_rate 来自 env.commands

输出：
- MAE
- RMSE
- max error

B. End-Effector Position Tracking Error
RoboDuet 使用 spherical coordinate 表达 EE target position，包括：
- l: radius，末端目标距离
- p: pitch / latitude
- y: yaw / longitude
并且还统计 Euclidean distance D。

在当前仓库中不一定使用 l,p,y 表示，因此请优先实现更通用的 3D Cartesian position error：

ee_pos_err = || actual_ee_pos - target_ee_pos ||_2

如果仓库中能找到 spherical command，则额外统计：
l_err, p_err, y_err。

输出：
- ee/pos_mean
- ee/pos_rmse
- ee/pos_median
- ee/pos_p90
- ee/pos_max

C. End-Effector Orientation Tracking Error
RoboDuet 使用 orientation tracking error，包括：
- alpha, beta, gamma 三个角度误差
- quaternion geodesic distance ζ

请优先实现 quaternion geodesic distance：

给定实际末端四元数 q_actual 和目标四元数 q_target：

dot = abs(sum(q_actual * q_target))
dot = clamp(dot, -1, 1)
theta = 2 * acos(dot)

theta 即 orientation error，单位 rad。

注意：
- 需要兼容 xyzw 和 wxyz 两种 quaternion 顺序。
- 优先检查仓库已有 quat 工具函数，例如 isaacgym.torch_utils 或 legged_gym 内部 quat 工具。
- 如果只有 rpy，没有 quaternion，则使用 wrapped angle error：
  angle_err = atan2(sin(actual - target), cos(actual - target))

输出：
- ee/ori_geodesic_mean
- ee/ori_geodesic_rmse
- ee/ori_geodesic_median
- ee/ori_geodesic_p90
- ee/ori_geodesic_max

D. Pose Success Rate
RoboDuet 认为一个 EE pose tracking sample 成功，需要同时满足：

1. 末端位置误差：
   D <= 0.03 m

2. 末端姿态误差：
   theta <= pi / 36

这里 theta 是 orientation error，约等于 5 degree。

3. 机器人没有摔倒。
4. 没有 reset。
5. 如果有 self-collision / unsafe collision flag，则不能发生碰撞。

请实现：
success = (ee_pos_err <= 0.03) & (ee_ori_err <= pi/36) & alive_mask

输出：
- ee/success_rate_step
- ee/success_rate_episode
- ee/time_to_success_mean
- ee/time_to_success_median

如果 play.py 中目标是连续 tracking，不是单目标 reaching，则统计所有 step 的 success rate。
如果可以识别 episode，则统计 episode 内是否曾经达到 success。

E. Survival Rate
RoboDuet 的 survival rate 是在扰动下统计机器人是否保持站立。论文中使用 10N 到 20N 的随机外力施加到 base 上持续 2 秒，然后看机器人是否保持 base height > 0.26 m。

当前 play 阶段可以先不强制实现外力扰动，但至少实现普通 rollout 下的 survival rate：

fall 条件：
- base_height < termination height
- 如果 cfg 里有 termination_height，优先使用 cfg
- 否则默认 fall_height_thr = 0.26
- 或者 env.reset_buf == 1 / env.termination_buf == 1

输出：
- stability/survival_rate_step
- stability/fall_rate_step
- stability/episode_survival_rate
- stability/base_height_mean
- stability/base_height_min
- stability/base_height_p05

如果仓库中方便施加外力，可以额外实现 optional perturbation eval：
--eval_perturb
--perturb_force_min 10
--perturb_force_max 20
--perturb_duration 2.0

但这个不是第一优先级，不能影响普通 eval。

F. Solvability
RoboDuet 中 solvability 定义为：

solvability = collision-free reachable poses / total sampled poses

含义：
目标 EE pose 从当前 pose 出发，在 tracking 过程中能够达到，并且不发生碰撞，则认为该 sample solvable。

在 play 阶段近似实现：
如果一个 target 在限定时间内满足：
- ee_pos_err <= 0.03
- ee_ori_err <= pi/36
- no fall
- no reset
- no self-collision / unsafe collision if available

则该 target 是 solvable。

输出：
- workspace/solvability_rate

如果 play.py 里没有明确 target 分段，则把每个 episode 作为一个 sample，或者把连续 tracking 中每个 env 的 target command 当作 sample。

G. Workspace
RoboDuet 中 workspace 是：
在 tracking error threshold 内成功达到、并且 tracking 过程中没有 self-collision 的目标命令点形成的 convex hull 面积/体积。

请实现近似版本：
收集所有 success 时刻的 target_ee_pos 或 actual_ee_pos。
如果点数足够：
- 3D convex hull volume: workspace/hull_volume
- x-z 平面 convex hull area: workspace/hull_area_xz
- x-y 平面 convex hull area: workspace/hull_area_xy

使用 scipy.spatial.ConvexHull。
如果 scipy 不存在，不要强制安装，跳过该指标并 warning。

输出：
- workspace/success_points
- workspace/hull_volume
- workspace/hull_area_xz
- workspace/hull_area_xy

------------------------------------------------------------
2. Deep Whole-Body Control 类指标
------------------------------------------------------------

Deep Whole-Body Control 主要评估 unified whole-body policy 相比 separate policy / uncoordinated policy 是否更协调。它报告的核心指标包括：

A. Survival Percentage
含义：
episode 内机器人保持稳定、不摔倒、不提前 reset 的比例。

这和上面的 survival rate 一致。

B. Base Acceleration
论文中使用 base angular acceleration 评估底盘是否稳定。base acceleration 越小，说明机械臂运动没有让底盘产生剧烈晃动。

实现：
base_ang_acc = (base_ang_vel_t - base_ang_vel_t_minus_1) / dt

如果有 base linear velocity，也统计：
base_lin_acc = (base_lin_vel_t - base_lin_vel_t_minus_1) / dt

输出：
- stability/base_ang_acc_mean
- stability/base_ang_acc_rms
- stability/base_ang_acc_p90
- stability/base_lin_acc_mean
- stability/base_lin_acc_rms
- stability/base_lin_acc_p90

C. Velocity Error
论文中 velocity error 是 base velocity command 和 actual base velocity 的 L1 error。

实现：
vel_l1 = abs(vx - vx_cmd) + abs(yaw_rate - yaw_rate_cmd)

如果有 vy command，则加入 vy：
vel_l1 = abs(vx - vx_cmd) + abs(vy - vy_cmd) + abs(yaw_rate - yaw_rate_cmd)

输出：
- vel/l1_mean
- vel/l1_rmse

D. EE Error
论文中 EE error 是 end-effector command 和 actual end-effector pose 的 L1 error。

请实现：
如果有 EE position：
ee_pos_l1 = sum(abs(actual_ee_pos - target_ee_pos))

如果有 EE orientation：
ee_ori_error 使用 quaternion geodesic distance 或 rpy wrapped error。

输出：
- ee/l1_mean
- ee/l1_rmse

E. Total Energy
论文中 total energy 是 legs + arm 的总能耗，且指标按 episode length 归一化。

如果有 env.torques 和 env.dof_vel：

power_abs = abs(torques * dof_vel)

需要统计：
- leg power
- arm power
- total power

如果能区分 leg dof 和 arm dof：
leg_energy = sum over leg joints and time of abs(tau * qdot) * dt
arm_energy = sum over arm joints and time of abs(tau * qdot) * dt

输出：
- energy/leg_power_abs_mean
- energy/arm_power_abs_mean
- energy/total_power_abs_mean
- energy/leg_energy_sum
- energy/arm_energy_sum
- energy/total_energy_sum
- energy/total_energy_per_episode
- energy/total_energy_per_step

RoboDuet 的 leg energy reward 还使用了 squared power 形式：
sum(|tau_i * qdot_i|^2)

所以额外统计：
- energy/leg_power_squared_mean
- energy/arm_power_squared_mean
- energy/total_power_squared_mean

F. Arm Workspace
Deep Whole-Body Control 用 1000 个 sampled EE poses 的 convex hull volume 估计 arm workspace。

请实现和上面 workspace 类似的统计：
- 收集成功 tracking 的 EE target 或 actual EE pose
- 用 convex hull volume 表示 reachable workspace

输出：
- workspace/hull_volume

G. Whole-Body Coordination Correlation
Deep Whole-Body Control 的 real-world analysis 说明：
- 当 EE command pitch p 改变时，quadruped body pitch 会随之改变，说明腿部通过调整身体俯仰帮助机械臂 reach。
- 当 EE command yaw y 靠近机器人侧方时，quadruped body roll 会随之改变，说明腿部通过侧倾帮助机械臂 reach。
- 当目标更远，即 radius / length l 更大时，base pitch 变化更明显。

请把这个思想变成诊断性指标：

如果能获取 target_ee_pos 或 ee command：
- target_ee_x
- target_ee_y
- target_ee_z
或者 spherical command：
- l
- p
- y

如果能获取 base orientation：
- base_roll
- base_pitch
- base_yaw

统计 Pearson correlation：
- coordination/corr_target_ee_x_base_pitch
- coordination/corr_target_ee_z_base_pitch
- coordination/corr_target_ee_y_base_roll
- coordination/corr_ee_pos_err_base_ang_acc
- coordination/corr_arm_action_norm_base_ang_acc

注意：
这些 correlation 不是越大越好，而是 diagnostic metrics，用来判断是否存在 whole-body coupling。请在打印结果中标注 diagnostic。

H. Stability Under Large Arm Motion
为了评估 arm 大动作时 base 是否稳定，请实现：

arm_action_delta = || arm_action_t - arm_action_t_minus_1 ||

large_arm_motion_mask = arm_action_delta > percentile(arm_action_delta, 75)

在 large arm motion 子集上统计：
- base angular acceleration
- base linear acceleration
- EE position error
- velocity tracking error
- survival rate

输出：
- coordination/base_ang_acc_when_arm_large
- coordination/base_lin_acc_when_arm_large
- coordination/ee_pos_err_when_arm_large
- coordination/vel_err_when_arm_large
- coordination/survival_when_arm_large

这个指标能反映机械臂大幅运动时，四足底盘有没有失稳。
